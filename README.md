#SauceDemo – Manual QA Web Testing

This project presents a complete Manual QA testing portfolio for the SauceDemo web application. It demonstrates my practical skills in software quality assurance, test case design, bug reporting, and structured QA documentation, aligned with industry practices.

📌## Overview

SauceDemo is a demo e-commerce platform widely used for QA practice.
In this project, I performed end-to-end manual testing of key user flows:

Login authentication
Product listing & filtering
Add/Remove items from cart
Product detail validation
Checkout process

The project reflects a real-world QA workflow, showcasing my ability to test applications systematically and report defects professionally.

##🐞 Key Bugs Identified

##Some important issues discovered:

Cart does not support multiple quantities of the same item
Checkout validation behaves inconsistently
Add-to-cart buttons become unresponsive after navigation
State mismatch between product page and cart

All bugs are documented with proper QA standards in bug-reports.md.
